# Resources

## Overview
Static assets and resources used by the app including logos, images, and brand materials.

### Logo/
- Contains Engify brand logo and logo variations
- Used in IntroView, LoginView, and onboarding flows
- Supported in both light and dark modes
- See Logo/README.md for asset specifications

## When to Update

### Adding New Assets
1. Organize assets by category in appropriate subfolder
2. Follow naming conventions (e.g., `logo-full`, `logo-icon`)
3. Provide multiple scales (@1x, @2x, @3x) if raster images
4. Use vector formats (SVG, PDF) when possible for scalability
5. Document asset dimensions and usage

### Logo Updates
- Update all logo variations consistently
- Test in both light and dark theme
- Ensure logo scales properly at different sizes
- Verify logo meets brand guidelines

### Asset Best Practices
- Use vector formats for logos and icons
- Minimize raster image file sizes
- Include accessibility labels for images
- Keep assets organized and well-named
- Document color specifications for rebranding
